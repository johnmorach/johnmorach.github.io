Assembly

From March 2015 to May 2015 I learned assembly language by following the information found at Programmed Introduction to MIPS Assembly Language. This was a much different experience from what I had worked on before - Java abstracts a lot of the details that programmers need to be concerned with when working in assembly.

Beginnings

I started out working with the very basics. In assembly I had to operate directly with the different memory registers, performing operations such as or and add to work with individual registers and multiple registers. At this point, I had no way of writing to any console or providing any user-friendly output - the output was the values in the different memory registers.

Syscall

The syscall command in MIPS Assembly Language is a very important one, and this command was the next thing that I learned. The syscall command allows for the implementation of features such as input and output through a console, which are necessary for creating useful programs. Combining syscall and other things that I learned how to implement in assembly language, such as loops, allowed me to write much more complex and useful programs than I was able to create before.

Functions

Up to this point in the process of learning assembly language, all of my programs had been essentially one function, although there were different sections within this function. Learning the jal and jr commands were essential, as they allowed me to create separate functions for performing specific tasks. This is a crucial part of programming, so learning these commands was a major step in being able to create larger and better programs.

Floating Point Registers

Floating point numbers are useful for many tasks within programs, so learning how to use the floating point registers through the MIPS Assembly Language was another important lesson. Floating point numbers are applicable to a variety of tasks where simple integers do not work or are not easy to use.

Conclusions

By the end of the work that I did with the MIPS Assembly Language, I was able to write programs that could be useful and that mimicked some of the programs that I could write in other languages such as Java. Programming in assembly language requires the programmer to consider many more details that are abstracted in higher-level languages, but assembly language gives programmers more freedom than other languages.
